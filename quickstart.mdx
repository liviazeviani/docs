This guide walks you through the core TaskFlow workflow: create a project, add a task to it, list your tasks, and update a task's status. By the end, you'll have made a full round-trip through the API.

<Note>
  The API may take up to 60 seconds to respond on the first request if it has been idle. Subsequent requests will be fast.
</Note>

## Prerequisites

You'll need an API key to authenticate requests. For testing, use the sandbox key below:

```bash
tf-live-4a8f2c1e9b3d7e6a
```

Every request must include it in the `Authorization` header as a Bearer token. See [Authentication](/authentication/) for details.

<Steps>
  <Step title="Create a project">
    Every task must belong to a project, so start by creating one. Only `name` is required.

    <CodeGroup>

    ```bash curl
      curl -X POST https://test-writechoice.onrender.com/projects \
        -H "Authorization: Bearer tf-live-4a8f2c1e9b3d7e6a" \
        -H "Content-Type: application/json" \
        -d '{
          "name": "API Documentation Revamp",
          "description": "Rewriting all public-facing API docs for v2."
        }'
    ```

    </CodeGroup>

    You'll get back a `project_id` — save it, you'll need it for the next step.

    ```json Response
      {
        "project_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
        "name": "API Documentation Revamp",
        "description": "Rewriting all public-facing API docs for v2.",
        "created_at": "2025-05-12T10:30:00.000Z"
      }
    ```
  </Step>
  <Step title="Create a task">
    Using the `project_id` from the previous step, create a task. New tasks always start with a status of `pending`.

    <CodeGroup>

    ```bash curl
      curl -X POST https://test-writechoice.onrender.com/tasks \
        -H "Authorization: Bearer tf-live-4a8f2c1e9b3d7e6a" \
        -H "Content-Type: application/json" \
        -d '{
          "project_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
          "title": "Write introduction page",
          "description": "Draft the overview and getting started sections."
        }'
    ```

    </CodeGroup>

    ```json Response
      {
        "task_id": "f9e8d7c6-b5a4-3210-fedc-ba9876543210",
        "project_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
        "title": "Write introduction page",
        "description": "Draft the overview and getting started sections.",
        "status": "pending",
        "created_at": "2025-05-12T10:35:00.000Z",
        "updated_at": "2025-05-12T10:35:00.000Z"
      }
    ```

    <Tip>
      If you use a `project_id` that doesn't exist, this request returns a `404`. Make sure you created a project first.
    </Tip>
  </Step>
  <Step title="List tasks in the project">
    Retrieve every task that belongs to a project.

    <CodeGroup>

    ```bash curl
      curl https://test-writechoice.onrender.com/projects/a1b2c3d4-e5f6-7890-abcd-ef1234567890/tasks \
        -H "Authorization: Bearer tf-live-4a8f2c1e9b3d7e6a"
    ```

    </CodeGroup>

    ```json Response
      {
        "project_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
        "total": 1,
        "tasks": [
          {
            "task_id": "f9e8d7c6-b5a4-3210-fedc-ba9876543210",
            "project_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
            "title": "Write introduction page",
            "description": "Draft the overview and getting started sections.",
            "status": "pending",
            "created_at": "2025-05-12T10:35:00.000Z",
            "updated_at": "2025-05-12T10:35:00.000Z"
          }
        ]
      }
    ```
  </Step>
  <Step title="Update the task's status">
    Once you start working on a task, update its status to `in_progress`. You only need to send the fields you want to change.

    <CodeGroup>

    ```bash curl
      curl -X PATCH https://test-writechoice.onrender.com/tasks/f9e8d7c6-b5a4-3210-fedc-ba9876543210 \
        -H "Authorization: Bearer tf-live-4a8f2c1e9b3d7e6a" \
        -H "Content-Type: application/json" \
        -d '{
          "status": "in_progress"
        }'
    ```

    </CodeGroup>

    ```json Response
      {
        "task_id": "f9e8d7c6-b5a4-3210-fedc-ba9876543210",
        "project_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
        "title": "Write introduction page",
        "description": "Draft the overview and getting started sections.",
        "status": "in_progress",
        "created_at": "2025-05-12T10:35:00.000Z",
        "updated_at": "2025-05-12T11:00:00.000Z"
      }
    ```

    Valid values for `status` are `pending`, `in_progress`, and `done`.
  </Step>
  <Step title="Delete the task (optional)">
    When a task is no longer needed, delete it permanently.

    <CodeGroup>

    ```bash curl
      curl -X DELETE https://test-writechoice.onrender.com/tasks/f9e8d7c6-b5a4-3210-fedc-ba9876543210 \
        -H "Authorization: Bearer tf-live-4a8f2c1e9b3d7e6a"
    ```

    </CodeGroup>

    ```json Response
      {
        "message": "Task 'f9e8d7c6-b5a4-3210-fedc-ba9876543210' was successfully deleted."
      }
    ```

    <Warning>
      This action cannot be undone.
    </Warning>
  </Step>
</Steps>
